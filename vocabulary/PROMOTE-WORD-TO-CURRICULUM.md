# Promoting Dictionary Words to Curriculum

This guide explains how to move a dictionary-only word into the curriculum so it appears in lessons, exercises, and tests.

---

## Overview

The dictionary system has two layers:

| Layer | Location | Purpose |
|-------|----------|---------|
| **Core** (curriculum) | `shared/vocabulary/core/de/` | Words actively taught in lessons |
| **Dictionary** (extended) | `shared/vocabulary/dictionary/de/` | Generated — all words with enrichment |

The `curriculum` flag in dictionary banks is **derived**, not the source of truth. A word is "curriculum" if and only if it exists in the core bank files.

---

## Steps to Promote a Word

### 1. Add the word to the core bank

Add a full entry to the appropriate file in `shared/vocabulary/core/de/`:

| Word type | Core bank file |
|-----------|---------------|
| Noun | `nounbank.json` |
| Verb | `verbbank.json` |
| Adjective | `adjectivebank.json` |
| Article/determiner | `articlesbank.json` |
| Pronoun | `pronounsbank.json` |
| Phrase | `phrasesbank.json` |
| Number | `numbersbank.json` |
| Other (adverb, preposition, etc.) | `generalbank.json` |

**Example — promoting "Abfahrt" (noun, currently dictionary-only):**

```json
"abfahrt_noun": {
    "word": "Abfahrt",
    "genus": "f",
    "plural": "die Abfahrten",
    "cases": {
        "nominativ": {
            "intro": "5.1",
            "bestemt": "die Abfahrt",
            "ubestemt": "eine Abfahrt"
        }
    },
    "_id": "abfahrt_noun",
    "audio": "substantiv_abfahrt.mp3"
}
```

**Required fields by word type:**

| Type | Required fields |
|------|----------------|
| Noun | `word`, `genus`, `plural`, `cases` (with `intro`), `_id`, `audio` |
| Verb | `word`, `conjugations` (with `intro`), `_id`, `audio` |
| Adjective | `word`, `comparison` (if applicable), `_id`, `audio` |
| Other | `word`, `type`, `_id`, `audio` |

The `intro` field (e.g., `"5.1"`) determines which lesson introduces the word and drives CEFR derivation:
- Lessons 1.x–2.x = A1
- Lessons 3.x–5.x = A2
- Lessons 6.x+ = B1

### 2. Add the word ID to the curriculum manifest

Edit the appropriate curriculum manifest in `shared/vocabulary/curricula/`:

```
vocab-manifest-tysk1-vg1.json    (VG1 Tysk 1)
vocab-manifest-tysk2-vg1.json    (VG1 Tysk 2)
vocab-manifest-us-8.json         (US 8. klasse)
...
```

Add the word ID to the lesson's `words` array:

```json
"5.1": {
    "words": ["abfahrt_noun", ...existing words...],
    "features": ["grammar_present", ...]
}
```

### 3. Add translations

Add translations in the curriculum translation files (NOT the dict translation files):

**Norwegian** — `shared/vocabulary/translations/de-nb/nounbank.json`:
```json
"abfahrt_noun": {
    "translation": "avgang",
    "synonyms": [],
    "translationSynonyms": ["avreise"],
    "examples": [
        {
            "sentence": "Die Abfahrt ist um 10 Uhr.",
            "translation": "Avgangen er klokka 10."
        }
    ]
}
```

**English** — `shared/vocabulary/translations/de-en/nounbank.json`:
```json
"abfahrt_noun": {
    "translation": "departure",
    "synonyms": [],
    "translationSynonyms": ["leaving"],
    "examples": [
        {
            "sentence": "Die Abfahrt ist um 10 Uhr.",
            "translation": "The departure is at 10 o'clock."
        }
    ]
}
```

### 4. Generate audio

If the word doesn't have an audio file yet, generate one:

```bash
# Single word
node back-stage/generate-single-audio.js "die Abfahrt" substantiv_abfahrt.mp3

# Or generate all missing audio
node back-stage/generate-vocab-audio.js --lang=de
```

Audio files go in `shared/vocabulary/core/de/audio/`.

### 5. Rebuild the dictionary

```bash
npm run build:dictionary
```

This will:
- Set `curriculum: true` on the word (via `enrich-curriculum-words.js`)
- Skip it in Goethe import (since it now exists in core)
- Update frequency data
- Remove any dict-only translation (curriculum translation takes priority)
- Rebuild the search index

### 6. Verify

```bash
# Check the word appears as curriculum in the dictionary
node -e "
const d = require('./shared/vocabulary/dictionary/de/nounbank.json');
const w = d['abfahrt_noun'];
console.log(w.word, '| curriculum:', w.curriculum, '| cefr:', w.cefr);
"

# Test the API locally
curl http://localhost:8000/api/vocab/v2/lookup/de/abfahrt_noun
```

---

## Important Notes

- **Never edit dictionary bank files directly** — they are regenerated by the build pipeline
- **Curriculum translations always take priority** over dict translations in the API
- **The `intro` field is required** on at least one case/conjugation — it drives CEFR level
- **Audio is expected** — the tooltip system looks for the `audio` field
- **Run `npm run build:dictionary` after changes** to update the search index

---

## Audio Generation

The existing TTS script at `back-stage/generate-vocab-audio.js` generates audio from `core/` bank files. It reads all wordbanks and finds entries without audio files.

```bash
# Dry run — see what's missing
node back-stage/generate-vocab-audio.js --dry-run

# Generate missing audio (requires GOOGLE_CLOUD_API_KEY in .env)
node back-stage/generate-vocab-audio.js --lang=de

# Force regenerate all
node back-stage/generate-vocab-audio.js --force
```

**Note:** This script currently only reads from `core/` banks. Dictionary-only words don't get audio until promoted to curriculum or until the script is adapted to also read from `dictionary/` banks.

## Quick Checklist

- [ ] Word added to correct `core/de/{bank}.json` with `_id`, `audio`, and `intro`
- [ ] Word ID added to curriculum manifest (`curricula/vocab-manifest-*.json`)
- [ ] Norwegian translation added to `translations/de-nb/{bank}.json`
- [ ] English translation added to `translations/de-en/{bank}.json`
- [ ] Audio file generated and placed in `core/de/audio/`
- [ ] `npm run build:dictionary` run successfully
- [ ] API lookup returns the word with `curriculum: true`
