The script has three components:

Script must `delete entry.comparison` when applying (schema forbids comparison when `nicht_komparierbar: true`). Keep `declension: {}` placeholder (these still get positiv declension in Phase 8).

- **Suppletive irregulars (3):** `gut_adj` → `{ komparativ: 'besser', superlativ: 'best' }`, `viel_adj` → `{ komparativ: 'mehr', superlativ: 'meist' }`, `hoch_adj` → `{ komparativ: 'höher', superlativ: 'höchst' }`.

- **Umlaut irregulars (15+2):** `alt_adj` → `{ komparativ: 'älter', superlativ: 'ältest' }`, `arm_adj` → `ärmer/ärmst`, `gross_adj` → `größer/größt`, `hart_adj` → `härter/härtest`, `jung_adj` → `jünger/jüngst`, `kalt_adj` → `kälter/kältest`, `kurz_adj` → `kürzer/kürzest`, `lang_adj` → `länger/längst`, `scharf_adj` → `schärfer/schärfst`, `stark_adj` → `stärker/stärkst`, `warm_adj` → `wärmer/wärmst`, `krank_adj` → `kränker/kränkst`, `dumm_adj` → `dümmer/dümmst`, `klug_adj` → `klüger/klügst`, `nass_adj` → `nässer/nässest`, `gesund_adj` → `gesünder/gesündest`, `schwach_adj` → `schwächer/schwächst`.

- **Consonant+d/t cluster superlative exceptions:** These need `-est` suffix instead of `-st` because the cluster would be unpronounceable. Add to exceptions: `blind_adj` → `blinder/blindest`, `rund_adj` → `runder/rundest`, `mild_adj` → `milder/mildest`, `wild_adj` → `wilder/wildest`, `fremd_adj` → `fremder/fremdest`.

- **-er e-drop irregulars (2):** `teuer_adj` → `{ komparativ: 'teurer', superlativ: 'teuerst' }`, `sauer_adj` → `{ komparativ: 'saurer', superlativ: 'sauerst' }`.

**2. Rule engine function** `generateComparison(word)`:
- If word ends in `-el`: drop `e` → `stem + 'ler'` for komparativ, `word + 'st'` for superlativ (e.g., dunkel → dunkler/dunkelst).
- If word ends in sibilant (`sch`, `ss`, `st`, `tz`) or single sibilant char (`s`, `ß`, `z`, `x`): regular komparativ (`word + 'er'`) but superlativ uses `-est` (e.g., frisch → frischer/frischest).
- Default: `word + 'er'` / `word + 'st'`.

**3. processBank(filePath)** function:
- Read bank JSON, iterate all entries (skip `_metadata`).
- For each entry: check EXCEPTIONS table by `_id`. If match: apply override (delete comparison field for undeclinable/nicht_komparierbar, set flag, set comparison for irregulars). If no match: apply rule engine using `entry.word`.
- For old entries (no comparison field yet): add comparison object from rule engine.
- For new stubs (have `comparison: {}`): replace empty object with populated one.
- Update `_metadata.generatedAt`.
- Write back with `JSON.stringify(bank, null, 2) + '\n'`.
- Log counts: regular, excepted, undeclinable, nicht_komparierbar.
