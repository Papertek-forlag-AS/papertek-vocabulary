**Step 1: Add declension_type to noun.schema.json**

In `vocabulary/schema/noun.schema.json`, add `declension_type` to the `nounEntry.properties` object (after `weak_masculine`):

```json
"declension_type": {
  "type": "string",
  "description": "Declension pattern classification for pedagogical and data-processing use",
  "enum": ["strong", "weak", "mixed", "plural-only", "uncountable"]
}
```

Do NOT change any other schema property. Do NOT add `declension_type` to `required`. The field is optional (some entries may not have it until Plan 02 runs).

**Step 2: Create scripts/validate-nouns.js**

Create an ESM script (import/export — project convention from package.json type:module) following the pattern from `scripts/validate-adjectives.js`. The script must:

1. Import AJV 2020 from `ajv/dist/2020.js`
2. Read and compile `vocabulary/schema/core-word.schema.json` and `vocabulary/schema/noun.schema.json`
3. Get the validator for `https://papertek.no/schemas/vocabulary/noun.schema.json`
4. Read `vocabulary/core/de/nounbank.json`
5. Run validation
6. If valid: print `PASS: All N noun entries validate against schema` and exit 0
7. If invalid: print error count, then print all errors as JSON, and exit 1

Important: Use `{ strict: false, allErrors: true }` for AJV options — matching the existing validate-adjectives.js pattern.

**Step 3: Add validate:nouns to package.json scripts**

Add to the `scripts` section in `package.json`:
```json
"validate:nouns": "node scripts/validate-nouns.js"
```

Insert it after the existing `validate:audio` line (before `validate:all`).

**Step 4: Run and verify**

Execute `npm run validate:nouns` to confirm it runs. The pre-existing baseline is expected to be approximately 356 errors. Record the exact error count — this becomes the baseline for Plan 02 to compare against.
