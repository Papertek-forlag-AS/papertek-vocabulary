  <action>
Write and run a Node.js script (inline or temporary file) that performs the complete stub generation pipeline:

**Step 1 — Read inputs:**
- Read `.planning/phases/04-goethe-adjective-extraction/04-candidates.json` — extract the `candidates` array (259 entries, each with `word`, `_id`, `cefr`, `source`)
- Read `vocabulary/dictionary/frequency/de_50k.txt` — build a frequency rank map where line number (1-based) = rank. Format: each line has `word count` separated by whitespace; take first token as key. Use `rankMap[word] ?? 0` for missing words (11 words won't be found: alternativ, ausländisch, berufstätig, eckig, einheitlich, haltbar, interkulturell, pauschal, städtisch, stilistisch, virtuell).
- Read `vocabulary/core/de/adjectivebank.json` (106 existing entries + `_metadata`)
- Read `vocabulary/dictionary/de/adjectivebank.json` (106 existing entries + `_metadata`)

**Step 2 — Safety check:**
- Verify zero overlap: every candidate `_id` must NOT exist as a key in either bank. If any overlap found, abort with error listing the duplicates.
- Verify candidate count is 259. If not, abort with error.

**Step 3 — Construct core bank stubs:**
For each candidate, add to the core bank object:
```javascript
coreBank[c._id] = {
  word: c.word,
  _id: c._id,
  audio: `adjektiv_${c._id.replace('_adj', '')}.mp3`,
  comparison: {},
  declension: {}
};
```
Do NOT add `undeclinable`, `nicht_komparierbar`, or `type` fields — those are Phase 7's responsibility.

**Step 4 — Construct dictionary bank stubs:**
For each candidate, add to the dictionary bank object:
```javascript
dictBank[c._id] = {
  word: c.word,
  _id: c._id,
  audio: `adjektiv_${c._id.replace('_adj', '')}.mp3`,
  curriculum: false,
  cefr: c.cefr,
  frequency: rankMap[c.word] ?? 0,
  comparison: {},
  declension: {}
};
```

**Step 5 — Sort and reassemble both banks:**
- Extract `_metadata` from both bank objects
- Sort all remaining keys (entries) by key name using `a.localeCompare(b)` (ASCII sort on `_id` keys — deterministic, no locale dependency)
- Reassemble: `_metadata` first (at the top), then sorted entries
- Update `_metadata.generatedAt` to current ISO timestamp in both banks

**Step 6 — Write bank files:**
- Write `vocabulary/core/de/adjectivebank.json` with `JSON.stringify(output, null, 2) + '\n'`
- Write `vocabulary/dictionary/de/adjectivebank.json` with `JSON.stringify(output, null, 2) + '\n'`

**Step 7 — Update manifests:**
Count actual entries (not relying on old manifest values):
- Core manifest (`vocabulary/core/de/manifest.json`): Set `files.adjectivebank.json` to 365. Recalculate `totalWords` by summing all `files` values after the update. Update `generatedAt`.
- Dictionary manifest (`vocabulary/dictionary/de/manifest.json`): Set `files.adjectivebank.json` to 365. Recalculate `totalWords` by summing all `files` values after the update. Add 259 to `dictionaryOnlyWords` (currently 0, new value 259 — all new entries have `curriculum: false`). Leave `curriculumWords` unchanged (867). Update `generatedAt`.

**Step 8 — Print summary:**
Log: entry counts for both banks (should be 365), manifest totals, number of candidates processed, number of frequency-0 entries.
