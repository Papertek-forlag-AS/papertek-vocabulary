<task type="auto">
  <name>Task 2: Add exception flags and conditional validation rules</name>
  <files>vocabulary/schema/adjective.schema.json</files>
  <action>
Add two boolean flag properties to `adjectiveEntry.properties`:
- `"undeclinable": { "type": "boolean" }` — for adjectives that take no endings (lila, rosa, orange, cool)
- `"nicht_komparierbar": { "type": "boolean" }` — for absolute/non-comparable adjectives

Then add an `allOf` array at the `adjectiveEntry` level (as a sibling of `properties` and `required`). This `allOf` contains two independent `if`/`then` blocks:

**Rule 1: undeclinable flag**
```json
{
  "if": {
    "properties": { "undeclinable": { "const": true } },
    "required": ["undeclinable"]
  },
  "then": {
    "properties": { "declension": false }
  }
}
```
This forbids any `declension` data when `undeclinable` is true. The `required: ["undeclinable"]` prevents the rule from vacuously triggering when the flag is absent (Pitfall 1 from research).

**Rule 2: nicht_komparierbar flag**
```json
{
  "if": {
    "properties": { "nicht_komparierbar": { "const": true } },
    "required": ["nicht_komparierbar"]
  },
  "then": {
    "properties": {
      "comparison": false,
      "declension": {
        "type": "object",
        "properties": {
          "komparativ": false,
          "superlativ": false
        }
      }
    }
  }
}
```
This forbids `comparison` entirely AND forbids `komparativ`/`superlativ` within `declension` when `nicht_komparierbar` is true. Only `positiv` declension is allowed.

IMPORTANT: Do NOT use `additionalProperties: false` inside the `then` branches (Pitfall 2 from research — it sees all properties of the object and rejects valid entries).

Use the exact verified patterns from 05-RESEARCH.md (Pattern 2).
  </action>
