Create `scripts/build-search-index.js` — a full rebuild script for the search index. This should be reusable (user decision: "decide based on reusability").

**Approach:** The current search index has 3454 entries with an established compact field format. Reverse-engineer the exact format from the existing index, then rebuild all entries from the dict banks + translation files + core verbbank (for pp field).

**Step 1: Analyze existing index format.** Read the first 5 entries of each type (noun, verb, adjective, etc.) from the current search-index.json to understand the exact field set per type. The research documented these formats:

Verb entry fields: id, w, t("verb"), f, c, cur, vc, sep, pp(NEW), tr.nb, tr.en
Noun entry fields: id, w, t("noun"), f, c, cur, g, tr.nb, tr.en
Adjective entry fields: id, w, t("adjective"), f, c, cur, tr.nb, tr.en
Other types: follow the same base pattern

**Step 2: Build the script.** ESM script that:
1. Reads ALL dict banks: verbbank.json, nounbank.json, adjectivebank.json, generalbank.json (all from vocabulary/dictionary/de/)
2. Reads core verbbank (vocabulary/core/de/verbbank.json) for pp field lookup
3. Reads translation files from vocabulary/translations/de-nb/ and vocabulary/translations/de-en/ (curriculum translations), and vocabulary/translations/de-nb-dict/ and vocabulary/translations/de-en-dict/ (dict-only translations) — curriculum first, dict as fallback
4. For each entry across all dict banks, builds a compact index entry:
   - Base fields: id (key), w (word), t (type), f (frequency), c (cefr), cur (curriculum flag)
   - Type-specific fields:
     - Verbs: vc (verbClass.default), sep (verbClass.separable), pp (from core verbbank perfektum.participle — Claude's discretion: use bare participle e.g. "angefangen" not "hat angefangen")
     - Nouns: g (genus)
     - Adjectives: (no extra fields beyond base)
   - Translation: tr.nb (Norwegian), tr.en (English)
5. Sorts all entries alphabetically by id (Phase 10 decision: "Sort all index entries alphabetically by id")
6. Writes to vocabulary/dictionary/de/search-index.json as `JSON.stringify(entries, null, 2)`
7. Logs total entry count

**Step 3: Run the script** — `node scripts/build-search-index.js`

**Step 4: Verify output:**
- Total entries should be ~3454 (same as before, or slightly different if dict banks changed)
- Spot-check 5 verb entries have pp field
- Spot-check 5 noun entries have g field but no pp field (per user decision: no case hints in index)
- Verify tr.nb and tr.en are populated for known curriculum entries

**Important notes:**
- Do NOT add case hints to noun entries in the search index (user decision: "Do NOT add case hints to noun entries")
- Use bare participle for pp field (Claude's discretion from research: "Use bare participle ('angefangen'), not auxiliary form")
- Full rebuild guarantees no stale data (user decision: "Full rebuild of search index, not partial")
- Check for entries in the existing index that have fields not covered by the dict banks (e.g., expression entries from generalbank may have different fields) — preserve all existing field patterns per type
