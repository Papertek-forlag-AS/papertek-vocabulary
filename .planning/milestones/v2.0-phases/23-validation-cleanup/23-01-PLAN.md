  <action>
Create `scripts/validate-migration.js` as an ESM script (project uses `type: module`). The script validates all 4 Phase 23 requirements in order:

**Section 1 — VALID-01: Schema validation of all 8 merged banks**

- Load AJV 2020 (same pattern as validate-nouns.js: `import Ajv2020 from 'ajv/dist/2020.js'`)
- Load `vocabulary/schema/core-word.schema.json` as shared schema
- Load bank-specific schemas: `noun.schema.json`, `verb.schema.json`, `adjective.schema.json`
- For the 3 banks WITH dedicated schemas (nounbank, verbbank, adjectivebank): validate entries against their schema (strip `_metadata` first, same as existing validators do)
- For the 5 banks WITHOUT dedicated schemas (generalbank, articlesbank, pronounsbank, numbersbank, phrasesbank): validate each entry against `core-word.schema.json` individually. These banks use the generic core-word schema since they don't have specialized fields.
- Report: `VALID-01: X/8 banks pass schema validation (0 errors)`
- Track failures count.

**Section 2 — VALID-02: v1 data integrity (curriculum entries)**

- Use `execSync('git show 653b503:vocabulary/core/de/{bankFile}')` to extract pre-migration core bank data for each of the 8 bank files. The commit `653b503` is the v1.3 milestone completion (pre-v2.0 migration baseline).
- Load current merged bank from `vocabulary/banks/de/{bankFile}`
- Load `vocabulary/banks/de/manifest.json` to get curriculum IDs per bank
- For each bank:
  - Parse old core bank JSON (from git)
  - Get curriculum IDs from manifest.banks[bankName].ids
  - For each entry in the old core bank (skip `_metadata`):
    - Verify the entry exists in the current merged bank
    - Compare the entry field-by-field using JSON.stringify deep comparison. The merged bank may have ADDITIONAL fields from the dictionary bank (curriculum, cefr, frequency, tags) — these are expected additions, not corruption. So compare only the FIELDS that existed in the old core entry.
  - Report: per-bank pass/fail with count of entries checked
- Report: `VALID-02: All curriculum entries match pre-migration core data`

**Section 3 — VALID-03: v2 data integrity (all entries)**

- Use `execSync('git show 653b503:vocabulary/dictionary/de/{bankFile}')` to extract pre-migration dictionary bank data
- Load current merged bank from `vocabulary/banks/de/{bankFile}`
- For each bank:
  - Parse old dict bank JSON (from git)
  - For each entry in the old dict bank (skip `_metadata`):
    - Verify the entry exists in the current merged bank
    - Compare entry fields: for each field in the OLD dict entry, verify the same field with the same value exists in the current merged bank entry. The merged bank may have EXTRA fields from core (grammar data like conjugations, cases, declension); that is expected. Only compare fields that existed in the old dict entry.
  - Report: per-bank pass/fail with count of entries checked
- Report: `VALID-03: All dictionary entries match pre-migration dict data`

**Section 4 — BANK-06: Old directories removed**

- Check `fs.existsSync('vocabulary/core/de/')` — must be false (deleted in Phase 22-02)
- Check `fs.existsSync('vocabulary/dictionary/de/')` — must be false (deleted in Phase 21-02)
- Note: `vocabulary/core/es/` and `vocabulary/core/fr/` still exist — these are NOT part of the German v2.0 migration scope. `vocabulary/dictionary/` still has `frequency/`, `sources/`, `verb-classification-de.json` — these are reference data, not bank directories.
- Report: `BANK-06: Old German bank directories do not exist`
