## Tech Debt (Non-Blocking)

### Phase 17: API Fixes
- **v2 handler hardcodes feature ID strings** — The handler pushes `'grammar_presens'`, `'grammar_preteritum'`, etc. as string literals (lines 238-281) rather than reading `vocabulary/grammar-features.json` at runtime. The v1 grammarfeatures endpoint reads the file as single source of truth. IDs match now but can drift silently if grammar-features.json is updated without a parallel code change to the v2 handler.

### Phase 19: Curriculum Manifest Fix
- **Stale ID in developer guide** — `vocabulary/PROMOTE-WORD-TO-CURRICULUM.md` line 86 contains `"grammar_present"` in a JSON example for promoting German words. Out of Phase 19 scope (which targeted `api/vocab/README.md` only), but developers following this guide will add the stale feature ID.

### Phase 16: Data Fixes
- **No manifest count validator in validate:all** — `scripts/fix-manifest-counts.js` exists as a one-off health check but is not part of the `validate:all` chain. Manifest counts can drift again without automated enforcement.
- **531 dict-only verbbank entries remain untyped** — Pre-existing condition, explicitly out of DATA-02 scope ("all 148 core verbbank entries"). These dict-only entries had no type field before v1.3 and still don't.

### Total: 4 items across 3 phases
